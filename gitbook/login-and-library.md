# 首次登录与音乐库配置

## 第一次进入应用

首次进入 Echoes 时，会看到一个两步登录流程：

1. 服务器地址
2. 认证信息

## 第一步：填写服务器地址

你需要填写：

- 服务器地址
- 音乐库名称，可选
- 线路名称，可选

### 服务器地址要求

- 必须填写完整 URL
- 支持 `http://` 和 `https://`
- 推荐优先使用 `https://`

例如：

- `https://music.example.com`
- `http://192.168.1.10:4533`

如果你填写的是 HTTP，客户端会弹出安全提示。这是当前实现里主动做的风险提醒，不是报错。

### 音乐库名称

这是客户端本地显示名，例如：

- 家庭 NAS
- 公网入口
- 私人曲库

不填也可以，客户端会使用服务器类型作为默认名称。

### 线路名称

这是当前地址的标签，例如：

- 主线路
- 家里
- 公网
- Tailscale

如果你后面要维护多地址，这个名字很有用。

## 第二步：填写认证信息

客户端会先检测服务器能力，然后再进入认证页。

如果识别到服务器支持 API Key，界面会优先展示：

- 用户名
- API Key
- 密码

推荐优先使用 API Key。

## 登录成功后会保存什么

本地音乐库配置里会保存：

- 音乐库名称
- 用户名
- 认证方式
- 当前地址列表
- 服务器类型和版本信息
- 是否支持 OpenSubsonic
- Embed Service 配置

## 多音乐库管理

在侧边栏顶部可以展开音乐库列表，进行：

- 切换当前音乐库
- 编辑现有音乐库
- 新增音乐库

适合以下场景：

- 一个人维护多套 Navidrome
- 同一台客户端登录不同服务器
- 一套音乐库配多条访问线路

## 多地址与线路切换

每个音乐库下可以维护多个地址。

支持的操作包括：

- 新增地址
- 编辑地址
- 删除地址
- 拖拽调整优先级
- 手动切换线路
- 自动选择最佳线路

### 地址的约束

新增地址时，客户端会校验它是否仍然指向同一个服务器。

换句话说：

- 一个音乐库下的多条地址必须对应同一份音乐内容
- 不能把两个不同 Navidrome 服务器混进同一个音乐库里

## 自动回退

在设置页里可以开启“线路自动回退”。

含义是：

- 你手动锁定了一条线路
- 如果这条线路不可用
- 客户端可以自动切到其他可用线路

这对“内网优先，外网兜底”的部署方式很实用。

## 配置 Embed Service

进入“编辑音乐库”后，可以看到 `Embed Service 离线下载` 配置区。

你需要填写：

| 字段 | 说明 |
| --- | --- |
| Embed Service URL | Embed Service 的访问地址 |
| API Key | Embed Service 的鉴权密钥 |
| Library ID | 当前默认使用 `default` |

填完后建议先点一次“测试连接”。

测试通过后，远程探索页和试听播放器里的“添加到离线下载队列”功能才能正常工作。
